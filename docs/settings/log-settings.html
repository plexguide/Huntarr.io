Log Settings

Configure logging levels, rotation, and cleanup for Huntarr.

Overview

  • Huntarr maintains both file logs and a database log
  • Configure verbosity, retention, and cleanup behavior
  • View log storage usage and clean up manually

Log Levels

LevelDescription
DEBUGVery detailed, useful for troubleshooting
INFOStandard operational messages (default)
WARNINGPotential issues that don't stop operation
ERRORProblems that need attention

Settings

SettingDefaultDescription
Log LevelINFOMinimum severity to log
Log Retention Days30Days before old log files are deleted
Max Entries per App10000Maximum database log entries per app
Auto CleanupEnabledAutomatically clean up old logs
Max Log File Size5 MBSize before log file rotation
Max Log Files5Number of rotated log files to keep

Log Storage

The Log Settings page shows current storage usage:

  • Total file log size and file count
  • Database log size
  • Total database entries

Manual Actions

  • Clean Up Now: Run log cleanup immediately (respects retention settings)
  • Clear All Logs: Remove all log entries from the database
Clearing all logs

Clearing all logs removes diagnostic history. Use "Clean Up Now" for routine maintenance and "Clear All Logs" only when needed.

Tips

  • Use DEBUG level when troubleshooting, then switch back to INFO
  • Set reasonable retention (7-30 days) to manage disk space
  • Monitor the storage stats periodically
  • Database logs grow faster than file logs — the max entries setting helps
