First Steps

You've installed Huntarr and completed setup. Here's what to do next.

Your Dashboard

  • The home page shows all connected apps and their hunt status
  • Grid view shows app cards with stats (missing, upgrading, completed)
  • List view shows detailed hunt activity entries
  • Toggle between views with the icons in the top-right of the Hunt Activities section

Connect All Your Apps

If you only connected one app during setup, add the rest now:

  1. Click on "Apps" in the sidebar
  2. Select the app type (Sonarr, Radarr, Lidarr, etc.)
  3. Click "Add Instance"
  4. Fill in the URL and API key
  5. Test and save
Multiple instances

Huntarr supports multiple instances of the same app. For example, you can have separate Sonarr instances for 1080p and 4K libraries.

Configure Schedules

By default, Huntarr checks all apps on a regular cycle. Customize this:

  1. Go to Settings > Scheduling
  2. Select the app and instance
  3. Set the cycle interval (how often to hunt)
  4. Set caps (max items per cycle) to avoid overloading indexers

Recommended starting configuration:

SettingRecommendedDescription
Hunt Interval30 minutesTime between hunt cycles
Missing Items per Cycle5How many missing items to search per cycle
Upgrade Items per Cycle3How many items to attempt upgrading per cycle

Set Up Notifications

  1. Go to Settings > Notifications
  2. Click "Add Connection"
  3. Choose your provider (Discord, Telegram, etc.)
  4. Configure credentials
  5. Send a test notification
  6. Choose which apps/instances trigger this notification

Explore Movie Hunt (Optional)

Movie Hunt is Huntarr's standalone movie grabber — it works independently of Radarr:

  1. Navigate to Movie Hunt in the sidebar
  2. Add indexers (NZB or Torrent)
  3. Add a download client (SABnzbd, qBittorrent, etc.)
  4. Browse, search, and grab movies directly

Tips for Success

  • Start small: Begin with low caps (3-5 items per cycle) and increase gradually
  • Check logs: If hunts aren't working, check Settings > Logs for detailed output
  • Multiple instances: Use separate schedules for different instances to spread the load
  • Backup regularly: Enable automatic backups under Settings > Backup & Restore
