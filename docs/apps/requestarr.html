Requestarr Documentation

What is Requestarr? Requestarr is Huntarr’s built-in discovery and request feature. Users browse movies and TV shows (via TMDB), request items, and Huntarr adds them to your configured Radarr and Sonarr instances. It uses your existing *arr instances—no separate app to install.

Requestarr uses true multi-instance support: you can have multiple Radarr and Sonarr instances in Huntarr. Requestarr lets you choose which instance to use when adding a movie or TV show, and you can set default instances and root folders in Requestarr Settings.

Overview

Views

  • Discover (Home): Browse trending and popular movies and TV from TMDB. Pick a default movie instance (Radarr) and default TV instance (Sonarr) in Requestarr Settings so the home view loads the right library.
  • Movies: Search and filter movies, then add them to a Radarr instance of your choice.
  • TV: Search and filter TV shows, then add them to a Sonarr instance of your choice.
  • Hidden Media: List of movies and TV shows you’ve hidden from discovery. You can unhide items here to make them requestable again.
  • Settings: Default instances, default root folders, discovery filters (region, providers, languages), request cooldown, and blacklisted genres.

Multi-Instance Behavior

Huntarr supports multiple Radarr and Sonarr instances. Requestarr works with all of them:

  • When adding a movie, you select which Radarr instance (and optionally root folder) to use.
  • When adding a TV show, you select which Sonarr instance (and optionally root folder) to use.
  • In Requestarr → Settings you can set Default Discovery Instances (default movie instance, default TV instance) so the Discover home view uses the right instance.
  • You can set Default Root Folders per app (Radarr, Sonarr) so new requests go to the correct disk when your first root folder isn’t the one you want.

Requestarr Settings

Available under Requestarr → Settings in the UI:

Setting Description
Default Movie Instance Radarr instance used when loading the Discover home view. If unset, the first available instance is used.
Default TV Instance Sonarr instance used when loading the Discover home view. If unset, the first available instance is used.
Default Root Folder (Radarr) Root folder used when adding movies via Requestarr. If unset, the first root folder in the selected Radarr instance is used.
Default Root Folder (Sonarr) Root folder used when adding TV shows via Requestarr. If unset, the first root folder in the selected Sonarr instance is used.
Region Region used for streaming availability (e.g. United States). Affects which providers and titles appear in discovery.
Providers Streaming providers to filter by (based on selected region).
Language(s) Original language(s) to filter discovery results.
Request Cooldown How long users must wait before requesting the same item again (e.g. 24 hours, 7 days). Optional “Reset All Cooldowns” for cooldowns over 25 hours.
Blacklisted Genres (TV / Movies) Genres hidden from the Movies and TV filter dropdowns so users cannot request by those genres.

Requirements

  • At least one Radarr instance configured in Huntarr (for movies).
  • At least one Sonarr instance configured in Huntarr (for TV).
  • No separate “Requestarr” app; Requestarr is a feature inside Huntarr that uses your existing instances.
