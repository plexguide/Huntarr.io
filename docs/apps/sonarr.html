Sonarr

Automatically hunt for missing TV episodes and upgrade existing ones.

Overview

  • Sonarr manages TV series — Huntarr connects to it to find missing episodes and upgrade quality
  • Works with Sonarr v3 and v4
  • Multiple Sonarr instances supported (e.g., one for 1080p, one for 4K)

Configuration

Instance Settings

Same as general app settings (URL, API key, name)

Hunt Settings

SettingDefaultDescription
Hunt MissingEnabledSearch for episodes without files
Hunt UpgradesEnabledUpgrade episodes below cutoff quality
Missing Items per Cycle1Episodes to search per hunt cycle
Upgrade Items per Cycle1Episodes to try upgrading per cycle
Monitored OnlyEnabledOnly hunt monitored series/episodes

How Hunting Works

  1. Huntarr queries Sonarr's API for missing episodes
  2. Filters by monitored status and availability
  3. Triggers a search command for each selected episode
  4. Sonarr handles the actual download via its indexers and clients
  5. Process repeats on the configured schedule

Tips

  • Start with low caps (1-3 per cycle) to avoid indexer rate limits
  • Use separate instances for different quality profiles
  • Check Sonarr's Activity queue if searches seem slow
  • Ensure Sonarr's indexers are healthy before enabling hunts
Huntarr triggers searches through Sonarr — make sure Sonarr has working indexers and a download client configured.
