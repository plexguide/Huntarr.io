Swaparr

Automatically manage stalled and slow downloads.

Overview

  • Swaparr monitors your download clients for stalled or slow downloads
  • When a download is stuck, Swaparr can automatically swap it for an alternative
  • Reduces manual intervention in managing your download queue
  • Works alongside Huntarr's hunting system

How It Works

  1. Swaparr monitors active downloads in your configured clients
  2. Detects downloads that are stalled, very slow, or stuck
  3. Removes the problematic download
  4. Triggers a new search for the same media through the appropriate *arr app
  5. A new, hopefully better, download begins automatically

Configuration

Instance Settings

Standard: URL, API key, name.

Swaparr Settings Table

SettingDefaultDescription
EnabledYesEnable Swaparr monitoring
Strike Count3Number of strikes before a download is swapped
Strike Interval5 minutesTime between strike checks
Max Download Speed—Downloads below this speed get strikes (KB/s)
Ignore Categories—Comma-separated categories to exclude

Dashboard Stats

Huntarr shows Swaparr statistics on the home page:

  • Total swaps performed
  • Active strikes
  • Downloads being monitored
  • Success rate

Tips

  • Set strike count to 3+ to avoid swapping temporarily slow downloads
  • Use ignore categories for large downloads that are expected to be slow
  • Monitor the Swaparr stats on the dashboard to see if settings need tuning
  • Works best with torrent clients (qBittorrent, Deluge, Transmission)
Swaparr removes and re-searches downloads. Make sure your *arr apps have good indexer coverage so alternative downloads can be found.
