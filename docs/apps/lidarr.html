Lidarr Documentation

What is Lidarr? Lidarr is an automated music downloading and management application. It monitors for new albums, searches indexers, and automatically downloads music using your preferred download clients.

Lidarr automates the process of monitoring and downloading music. It integrates with popular download clients and uses indexers to search for and download albums automatically, making it easy to build and maintain an organized music library.

Overview

What Lidarr Does

Lidarr provides automated content management for music by:

  • Automatic Searching: Monitors for new albums and searches configured indexers
  • Quality Management: Downloads content based on your quality preferences (MP3, FLAC, etc.)
  • Library Organization: Automatically organizes downloaded music in your media library
  • Upgrade Management: Can upgrade existing music to better quality when available
  • Download Client Integration: Works with popular download clients like qBittorrent, Deluge, etc.

How Huntarr Enhances Lidarr

Huntarr adds intelligent automation on top of Lidarr:

  • Smart Search Timing: Prevents overwhelming indexers with too many requests
  • Missing Album Detection: Identifies and prioritizes missing albums and tracks
  • Upgrade Intelligence: Finds better quality versions of existing music
  • Rate Limiting: Respects API limits to prevent bans or throttling
  • Multi-Instance Support: Manage multiple Lidarr instances from one interface

Instance Configuration

Configure your Lidarr instances to connect Huntarr with your Lidarr installation(s).

True multi-instance
Huntarr supports multiple Lidarr instances. Each instance has its own connection (URL, API key), and its own per-instance settings: missing/upgrade mode (album vs artist), upgrade selection method, tags, search limits, and Swaparr. You can enable or disable each instance independently.

Adding Your Lidarr Instance

To connect your existing Lidarr instance to Huntarr:

  1. Navigate to the Lidarr section in Huntarr
  2. Click "Add Lidarr Instance" if you don't have one configured
  3. Fill in your Lidarr connection details
  4. Test the connection to ensure Huntarr can communicate with Lidarr

These settings match the Search Settings section in the instance editor. Each Lidarr instance has its own values.

Search counts

  • Missing Search Count ‚Äî Number of missing items to search for in each cycle. Set to 0 to disable missing searches.
  • Upgrade Search Count ‚Äî Number of items to upgrade in each cycle. Set to 0 to disable upgrade searches.

Missing Search Mode

Determines how Huntarr searches for missing music. In the UI this is typically Album (search for individual albums; Artist mode was deprecated in Huntarr 7.5.0+).

Upgrade Mode

Controls how Huntarr handles quality upgrades:

  • Albums: Upgrades entire albums when better quality becomes available
  • Tracks: Upgrades individual tracks for better quality
  • Artists: Upgrades at the artist level for consistent quality across discography

Upgrade Selection Method

You can choose how Huntarr selects which artists to consider for upgrades:

  • Cutoff unmet (default): Uses Lidarr's Wanted ‚Üí Cutoff Unmet list. Only albums that don't meet your quality/custom-format cutoff are eligible. Same behavior as before.
  • Tags (Upgradinatorr-style): Huntarr finds artists that DON'T have your specified tag (e.g. upgradinatorr), searches their albums for upgrades, then ADDS the tag to the artist after processing to mark them as complete. This matches Upgradinatorr's behavior: the tag tracks which artists have already been processed, not which ones to search. The tag is automatically created in Lidarr if it doesn't exist.

In both cases, Huntarr triggers AlbumSearch for the selected albums; Lidarr still decides what to grab based on your quality profile and custom format scores. When using Tags, the Upgrade Tag field sets the tag name on artists in Lidarr (e.g. upgradinatorr); Huntarr finds artists without this tag, runs upgrade searches on their albums, then adds the tag when done.

Cutoff unmet mode uses Lidarr's Wanted ‚Üí Cutoff Unmet list every cycle; Tags (Upgradinatorr) mode finds artists without your chosen tag, runs upgrade searches on their albums, then adds that tag to the artist so they are not searched again. Custom formats and quality profiles in Lidarr (and Trash Guides for Lidarr where available) determine what counts as an upgrade when Huntarr triggers the search.

Cutoff Unmet vs Tags: Which Should I Use?

Understanding the difference between these two modes will help you choose the right approach for your library:

How It Works Queries Lidarr's Wanted ‚Üí Cutoff Unmet list every cycle Searches albums by artists WITHOUT the tag, then ADDS tag to artist after processing
Re-processing Same albums appear in every cycle until they meet cutoff Once tagged, artist's albums are NEVER searched again (unless you manually remove tag)

Indexer Impact Higher - searches same albums repeatedly if upgrades not found Lower - each artist's albums searched once, then marked complete

üìä Example Scenario:
Cutoff Unmet: You have 200 albums below cutoff from 50 artists. Huntarr searches 5 albums per cycle, finds upgrades for 3. Next cycle, those 3 are gone (met cutoff), but the other 2 appear again along with 3 new ones from the remaining 195. This continues until all 200 meet cutoff or no more upgrades exist.

Tags: You have 1000 artists with no upgradinatorr tag. Huntarr searches 5 albums per cycle (from different artists), tags those artists after processing (whether upgrades found or not). Next cycle searches albums from 5 different artists. Eventually all 1000 artists have been processed once and tagged - done forever.

Custom Format Scores & Quality Upgrades

üìã Lidarr Configuration Required:
  • Lidarr ‚Üí Settings ‚Üí Profiles ‚Üí Quality Profile ‚Üí Upgrade Until Custom Format Score
  • Set Upgrade Until Custom Format Score to at least 10000

Why Custom Format Scores Matter

Huntarr does not decide what to upgrade‚ÄîLidarr does. When Huntarr triggers a search for upgrades, Lidarr evaluates the search results using your configured quality profile and custom format scores to determine whether a release is an upgrade worth grabbing.

Key Points:

  • Quality Cutoff: The minimum quality level you're willing to accept (e.g., MP3-320, FLAC)
  • Custom Formats: Additional scoring criteria beyond basic quality (e.g., preferred release groups, audio codec preferences, edition types)
  • Upgrade Until Custom Format Score: The score threshold at which Lidarr stops considering upgrades. Setting this to 10000 (or higher) ensures Lidarr will continue upgrading as long as better-scoring releases are found

How It Works:

  1. Huntarr identifies albums eligible for upgrade (based on your chosen upgrade selection method)
  2. Huntarr triggers an AlbumSearch command in Lidarr
  3. Lidarr searches indexers and scores each result using your quality profile + custom formats
  4. Lidarr automatically grabs releases that score higher than your current file (up to your "Upgrade Until" threshold)
  5. Without proper custom format scoring configured, Lidarr may not grab upgrades even when better releases are available
üí° Tip: See TrashGuides for comprehensive custom format scoring guides, recommended release group scores, and best practices for quality upgrades in Lidarr.

Additional Options

Fine-tune Huntarr's behavior with these additional settings:

Monitored Only

When enabled, Huntarr will only search for music that is marked as "Monitored" in Lidarr. This prevents searches for artists or albums you've intentionally unmonitored.

Skip Future Releases

When enabled, Huntarr will skip albums with future release dates. This prevents unnecessary searches for content that hasn't been released yet.

Quality Profiles

Respects Lidarr's quality profile settings. Huntarr will only search for music that meets your configured quality requirements (MP3, FLAC, etc.).

Artist Monitoring

When enabled, Huntarr will respect your artist monitoring settings in Lidarr, only searching for albums from artists you're actively monitoring.

Advanced Settings

These match the Advanced Settings section in the instance editor: API Timeout (seconds), Command Wait Delay (seconds), Command Wait Attempts, Max Download Queue Size, and Swaparr Monitoring (when Swaparr is enabled in Huntarr Settings).

Custom Tags

These match the Custom Tags section. Huntarr can add tags in Lidarr to mark processed items.

  • Tag missing items / Missing Items Tag ‚Äî Tag (e.g. huntarr-missing) added when items are found by a missing search. Max 25 characters.
  • Tag upgrade items / Upgrade Items Tag ‚Äî Used only in cutoff unmet mode. In Tags (Upgradinatorr) mode this option is hidden; Huntarr uses the Upgrade Tag (e.g. upgradinatorr) on artists to mark processed content instead.

Lidarr does not have "Tag shows missing" (Sonarr-only).

Exempt Tags

Items with any of these tags are skipped for missing and upgrade searches. If you remove the tag in Lidarr, Huntarr will process the item again. The tag upgradinatorr cannot be added as an exempt tag. See #676.

Swaparr Integration

Enable Swaparr monitoring for this Lidarr instance to automatically clean up stalled downloads.

What is Swaparr? Swaparr monitors your download queues and automatically removes stalled, failed, or problematic downloads to keep your automation running smoothly. Learn more about Swaparr configuration.

When to Enable Swaparr

  • You frequently have stalled music downloads
  • Your download queue gets clogged with failed album downloads
  • You want hands-off automation for your music library
  • You use public trackers where quality varies
  • You download large album collections that sometimes fail

When to Be Cautious

  • You use private music trackers with strict ratio requirements
  • You prefer manual control over your music downloads
  • You have very slow internet where downloads take a long time
  • You're building ratio on specific music torrents
  • You collect rare or hard-to-find music that shouldn't be removed

Troubleshooting

Common Issues

Connection Failed

If Huntarr can't connect to your Lidarr instance:

  • Verify the URL is correct and includes the protocol (http/https)
  • Check that the port number is correct (default: 8686)
  • Ensure the API key is copied correctly from Lidarr Settings > General > Security
  • Confirm Lidarr is running and accessible from Huntarr's network
  • Check firewall settings if Huntarr and Lidarr are on different machines

No Missing Music Found

If Huntarr reports no missing music but you know there are gaps:

  • Ensure "Monitored Only" is configured correctly for your needs
  • Check that artists and albums are marked as "Monitored" in Lidarr
  • Verify "Skip Future Releases" isn't preventing searches for available content
  • Review your quality profile settings in both Lidarr and Huntarr

Music Indexer Rate Limiting

If you're getting rate limited by music indexers:

  • Reduce the "API Cap - Hourly" setting significantly (try 100-200)
  • Increase the "Sleep Duration" to 30+ minutes between cycles
  • Lower the "Missing Search" and "Upgrade Search" limits to 1-2
  • Music indexers are typically more restrictive than movie/TV indexers

Metadata Issues

If Huntarr isn't finding the right music:

  • Verify your metadata provider settings in Lidarr (MusicBrainz, etc.)
  • Check that album and artist names are correctly matched
  • Review release group settings for proper album identification
  • Consider adjusting search criteria for international releases

Quality and Format Issues

If Huntarr isn't finding the right audio quality:

  • Verify your quality profiles support the formats you want (FLAC, MP3, etc.)
  • Check that your indexers have the quality levels you're seeking
  • Review cutoff settings to ensure upgrades are allowed
  • Consider separate instances for different quality requirements
